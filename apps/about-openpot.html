OpenPot

Version 1.0
Your agent, in your pocket. The native iOS command center for self-hosted AI agents.
Origin

OpenPot started with a simple frustration: powerful self-hosted AI agents trapped behind chat apps that were never designed for them. Telegram threads, Slack channels, terminal windows — none of them could show you what your agent knows, what it's done, or what you should do next.

"This app navigates the user so the user does not need to navigate the app."

OpenPot is built as the native iOS interface that self-hosted agents deserve. Not a wrapper around a web view. Not a chatbot skin. A real app — with purpose-built surfaces for every dimension of your relationship with your agent.

It connects to any agent running on the OpenClaw framework via secure WebSocket, with Ed25519 device authentication. Your data stays on your server. Nothing passes through a cloud service. The app talks directly to your agent over your network.

Deployment Tiers
Tier 1 Gateway Only
Connect to any OpenClaw gateway. Chat, Skills, Terminal, and Apps work immediately. Zero backend changes required. Five-minute setup.
Tier 2 Gateway + Server
Add PostgreSQL + pgvector for the memory layer. Pulse cards, Calendar integration, and chat persistence come online.
Tier 3 Full Stack
Agent operational cycle with adaptive cards, engagement scoring, and learning loop. The dashboard that learns from how you use it.
Getting Started

🔌 Connect

In the Agents tab, add your agent's gateway address. OpenPot connects via WebSocket on port 18789. Add both LAN and Tailscale routes for automatic failover.

📦 Install the Skill

Tell your agent: "Install the OpenPot Awareness Skill." Then run "OpenPot sync." Your agent configures itself for all the features it can support.

🗣️ Start Talking

Chat, use voice, ask your agent to build you an app, check your calendar, browse your Pulse cards. The app is your agent's home screen.
